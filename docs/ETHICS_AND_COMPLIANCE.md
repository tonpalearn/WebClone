# WebClone Ethics and Compliance

## Purpose

WebClone creates redesign demos as sales proposals. It must not impersonate a business, steal assets, or collect unnecessary personal data.

## Allowed

- Finding publicly listed businesses through normal web search
- Recording public business contact details such as business phone number, website, address, or contact form
- Summarizing visible website issues
- Creating a redesign concept using original layout/copy and permitted placeholder assets
- Sending outreach only after explicit user approval

## Not Allowed

- Scraping behind logins, CAPTCHAs, paywalls, or technical protections
- Ignoring robots/ToS restrictions for automated collection
- Collecting private personal data about staff/customers
- Copying proprietary images, logos, full text, menus, or designs without permission
- Creating a site that appears to be the official site without approval
- Publishing demos to confusing domains or using deceptive SEO
- Making fake claims, testimonials, certifications, awards, prices, or medical/legal guarantees
- Sending bulk unsolicited messages without user review and local-law compliance

## Required Demo Disclaimer

Every generated demo must include:

> Concept redesign demo prepared for discussion. This is not the official website unless approved by the business owner.

Thai version:

> ตัวอย่างคอนเซ็ปต์เว็บไซต์ที่ออกแบบเพื่อใช้พูดคุย/เสนอแนวทางเท่านั้น ยังไม่ใช่เว็บไซต์ทางการของธุรกิจนี้ เว้นแต่เจ้าของธุรกิจจะอนุมัติ

## Contact Rules

- Use business-level public contacts only
- Do not infer private phone numbers or personal emails
- Keep source URL for every contact field
- Ask user confirmation before sending outreach

## Asset Rules

- Use placeholders, generated neutral assets, or royalty-free assets
- If using real business logo/photo, require explicit approval or confirm license
- Avoid copying their current site's exact copy/design

## Publishing Rules

Before publishing any demo publicly:

1. Confirm with the user
2. Use a clearly demo/proposal URL
3. Include disclaimer
4. Avoid SEO indexing if the business has not approved
5. Do not use a domain that implies official ownership

## Recommended Outreach Positioning

Good:

> ผมทำ mockup หน้าเว็บเวอร์ชันใหม่ให้ดูเป็นตัวอย่างครับ ถ้าสนใจสามารถปรับให้เป็นเว็บจริงของร้านได้

Avoid:

> ผมทำเว็บของร้านคุณแล้ว / This is your new official website

## Review Checklist

- [ ] All contact details are public and source-backed
- [ ] No private personal data included
- [ ] No copied protected assets
- [ ] Demo disclaimer is visible
- [ ] Outreach is truthful and non-deceptive
- [ ] User approved any sending/publishing step
